# 🏃‍♀️ FitMeet  
**AI 기반 헬스케어 & 커뮤니티 통합 플랫폼**

---

## 📌 프로젝트 소개

FitMeet는 기존 헬스케어 서비스가 가진 한계인  
**수기 입력의 피로도**, **개인화 분석의 부재**, **데이터 단절 문제**를 해결하기 위해 기획된  
**AI 기반 헬스케어 통합 플랫폼**입니다.

사용자는 식단 / 운동 / 커뮤니티 데이터를 하나의 흐름으로 관리하고,  
AI 분석을 통해 **개인 맞춤형 건강 인사이트**를 제공받을 수 있습니다.

---

## ❗ 문제 인식 (Background)

### 기존 헬스케어 서비스의 한계

- 📉 **수기 입력 피로도**
  - 음식 중량, 칼로리 직접 입력의 번거로움
  - 사용자 78%가 2주 내 서비스 이탈

- 📊 **개인화 분석 부족**
  - 기초대사량(BMR), 활동량, 목표를 반영하지 않는 획일적 통계

- 🔄 **데이터 단절**
  - 식단, 운동, 커뮤니티 서비스가 분리되어 통합 인사이트 제공 불가

- 🔐 **보안 및 신뢰성 문제**
  - 민감한 신체 정보 및 이미지 데이터에 대한 보안 우려

## 🔑 핵심 기능

### 🥗 식단 관리
- **3가지 식단 등록 방식**
  - 직접 입력
  - 공공데이터 기반 음식 DB 검색
  - **AI Vision 이미지 분석**
- 섭취량 자동 스케일링 및 영양소 정규화
- 식단 점수 및 연속 기록(스트릭) 자동 계산

### 🤖 AI 분석 엔진
- OpenAI Vision 기반 음식 이미지 분석
- 영양 성분 추출 및 공공 데이터 매핑
- **주간 AI 식단 리포트 자동 생성**
  - 영양 패턴 요약
  - 개선점 및 맞춤 추천 제공

### 🏋️ 운동 & 추천
- YouTube API 연동 운동 영상 동기화
- 난이도/시간/부위 기반 필터링
- 콘텐츠 기반 + 사용자 행동 기반 **맞춤 추천**

### 👥 커뮤니티
- 식단 · 운동 기록과 연계된 게시글 작성
- 태그 기반 검색 및 필터링
- 좋아요, 댓글(대댓글), 조회수 관리
- Soft Delete 기반 게시글 관리

---

## 🔐 인증 & 보안

- **Spring Security + JWT 인증**
- Access / Refresh Token 로테이션
- BCrypt 비밀번호 단방향 해싱
- HttpOnly / Secure Cookie 적용
- Global Exception Handler 기반 표준 에러 처리

## 🧱 시스템 아키텍처

- **Frontend**
  - Vue 3, Vite, Pinia, TailwindCSS
- **Backend**
  - Spring Boot, MyBatis, Java 17
- **Database**
  - MySQL 8.0 (인덱스 최적화, 논리 삭제)
- **Infrastructure**
  - AWS EC2, S3, CloudFront
  - Presigned URL 기반 이미지 업로드
- **External APIs**
  - OpenAI Vision API
  - 공공데이터 포털
  - YouTube Data API
